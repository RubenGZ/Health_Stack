§1 Stack Tecnológico

Frontend
Next.js 14+ (App Router) — TypeScript

Server Components por defecto. Tailwind CSS + Shadcn/ui. React Query para data fetching. Zustand para estado global. Zod para validación de formularios del lado cliente.

Backend
FastAPI (Python 3.12) — Arquitectura de Capas

Routers → Services → Repositories → Models. SQLAlchemy 2.0 + Alembic para migraciones. Pydantic v2 para validación. Uvicorn + Gunicorn como servidor ASGI.

Base de datos
PostgreSQL 16 + Prisma ORM (también SQLAlchemy)

Dos esquemas separados: public (identidad) y health (biometría). Row-Level Security (RLS) activado. Prisma para el frontend, SQLAlchemy para el backend Python.

Infra
Hetzner CX31 — Fráncfort (EU-Central)

Docker Compose en producción. Nginx como reverse proxy. Certbot/Let's Encrypt para TLS 1.3. Backups diarios cifrados a Hetzner S3-compatible en Fráncfort.

Seguridad
AES-256-GCM + RSA-OAEP + Argon2id

Llave de cruce cifrada con AES-256-GCM. Contraseñas con Argon2id (no bcrypt). JWT RS256 (clave asimétrica). HSTS, CSP, X-Frame-Options en todas las respuestas.

Observabilidad
Sentry + Prometheus + Grafana + Loki

Sentry para error tracking (con data scrubbing de PII). Prometheus + Grafana para métricas de negocio y sistema. Loki para logs estructurados (sin datos de salud en logs).

§5 Llave de Cruce — AES-256-GCM (Paso 3)

El CryptoService es el único punto del sistema que puede vincular un user_id de la tabla de identidad con su health_subject_id en la base de datos de salud. Si alguien obtiene acceso a la tabla BiometricData, solo verá UUIDs opacos sin ninguna vinculación a nombres o emails.

🔑
Rotación de claves: La clave maestra AES-256 se almacena en variables de entorno nunca en la base de datos. Para rotación, se re-cifra el campo healthUuidEnc y se registra rotatedAt. Se recomienda usar HashiCorp Vault o AWS Secrets Manager para gestión avanzada de claves.
Python apps/api/app/services/crypto_service.py 
import os, uuid, binascii
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
from cryptography.exceptions import InvalidTag
from dataclasses import dataclass
import logging

logger = logging.getLogger("crypto_service")

@dataclass
class EncryptedPayload:
    """Payload cifrado: iv:auth_tag:ciphertext (hex separado por ':')"""
    iv: bytes
    auth_tag: bytes
    ciphertext: bytes

    def to_string(self) -> str:
        """Serializa para almacenar en DB como columna TEXT"""
        return ":".join([
            binascii.hexlify(self.iv).decode(),
            binascii.hexlify(self.auth_tag).decode(),
            binascii.hexlify(self.ciphertext).decode(),
        ])

    @classmethod
    def from_string(cls, payload: str) -> "EncryptedPayload":
        parts = payload.split(":")
        if len(parts) != 3:
            raise ValueError("Formato de payload cifrado inválido")
        return cls(
            iv=binascii.unhexlify(parts[0]),
            auth_tag=binascii.unhexlify(parts[1]),
            ciphertext=binascii.unhexlify(parts[2]),
        )


class CryptoService:
    """
    Servicio único para cifrar/descifrar la llave de cruce.
    PRINCIPIO: Si la tabla health.biometric_data es exfiltrada,
    health_subject_id no puede vincularse a ningún usuario sin
    pasar por este servicio con la clave maestra.
    """

    # Separación de contexto: evita que la misma clave se reutilice
    # en otros contextos de cifrado del sistema
    _CONTEXT = b"healthstack.health_link.v1"

    def __init__(self):
        raw_key = os.environ.get("HEALTH_LINK_MASTER_KEY")
        if not raw_key:
            raise EnvironmentError(
                "HEALTH_LINK_MASTER_KEY no está definida. "
                "Esta variable NUNCA debe estar en código fuente."
            )
        key_bytes = binascii.unhexlify(raw_key)
        if len(key_bytes) != 32:  # 256 bits
            raise ValueError("La clave maestra debe ser exactamente 256 bits (32 bytes)")
        self._aesgcm = AESGCM(key_bytes)

    def generate_health_subject_id(self) -> str:
        """Genera un UUID opaco para el sujeto de salud (no relacionado con user.id)"""
        return str(uuid.uuid4())

    def encrypt_health_link(self, health_subject_id: str) -> str:
        """
        Cifra el health_subject_id para almacenar en health_link.health_uuid_enc
        Usa AES-256-GCM con IV aleatorio de 96 bits (12 bytes)
        """
        iv = os.urandom(12)  # NIST recomendado: 96 bits para GCM
        plaintext = health_subject_id.encode("utf-8")

        # El AAD (Additional Authenticated Data) vincula el cifrado al contexto
        # Evita que un ciphertext de un registro sea reutilizado en otro
        ciphertext_with_tag = self._aesgcm.encrypt(iv, plaintext, self._CONTEXT)

        # GCM append el auth_tag (16 bytes) al final del ciphertext
        ciphertext = ciphertext_with_tag[:-16]
        auth_tag   = ciphertext_with_tag[-16:]

        payload = EncryptedPayload(iv=iv, auth_tag=auth_tag, ciphertext=ciphertext)
        return payload.to_string()

    def decrypt_health_link(self, encrypted_payload: str) -> str:
        """
        Descifra y devuelve el health_subject_id.
        Lanza InvalidTag si el payload fue manipulado.
        """
        try:
            payload = EncryptedPayload.from_string(encrypted_payload)
            ciphertext_with_tag = payload.ciphertext + payload.auth_tag
            plaintext = self._aesgcm.decrypt(
                payload.iv,
                ciphertext_with_tag,
                self._CONTEXT
            )
            return plaintext.decode("utf-8")
        except InvalidTag:
            # Log de seguridad SIN incluir el payload (evita fuga de datos cifrados)
            logger.error("CryptoService: InvalidTag — posible manipulación de datos")
            raise PermissionError("No se pudo verificar la integridad del enlace de salud")

    def resolve_health_subject(self, user_id: str, db_session) -> str:
        """
        API pública del servicio: dado un user_id,
        devuelve el health_subject_id para consultar datos de salud.
        
        FLUJO:
        1. Obtener health_link del usuario (tabla public.health_link)
        2. Descifrar health_uuid_enc
        3. Usar el UUID resultante para consultar health.biometric_data
        """
        from app.repositories.identity import HealthLinkRepository
        link = HealthLinkRepository.get_by_user_id(db_session, user_id)
        if not link:
            raise ValueError(f"No existe health_link para user_id={user_id}")
        return self.decrypt_health_link(link.health_uuid_enc)

    def create_health_link(self, user_id: str, db_session) -> str:
        """
        Crea el vínculo al registrarse el usuario.
        Genera health_subject_id, lo cifra y persiste el enlace.
        Devuelve el health_subject_id para crear el registro inicial en health schema.
        """
        from app.repositories.identity import HealthLinkRepository
        health_subject_id = self.generate_health_subject_id()
        encrypted = self.encrypt_health_link(health_subject_id)
        HealthLinkRepository.create(db_session, user_id=user_id, health_uuid_enc=encrypted)
        return health_subject_id


# Instancia singleton — se inyecta vía FastAPI Depends()
crypto_service = CryptoService()

§6 Strategy Pattern — Motor Nutricional (Paso 2)

El Strategy Pattern permite intercambiar el algoritmo de cálculo nutricional sin modificar la clase contexto NutritionCalculator. Para añadir una nueva dieta (ej: MediterraneanStrategy), solo se crea una nueva clase que implemente NutritionStrategy.

Python apps/api/app/services/nutrition/base_strategy.py 
from abc import ABC, abstractmethod
from dataclasses import dataclass
from enum import Enum


class FitnessGoal(Enum):
    BULK        = "bulk"
    CUT         = "cut"
    MAINTAIN    = "maintain"
    RECOMPOSE   = "recompose"


@dataclass
class UserBiometrics:
    weight_kg:      float
    height_cm:      float
    age:            int
    sex:            str   # "M" | "F"
    body_fat_pct:   float | None = None
    activity_level: float = 1.55   # TDEE multiplier
    goal:           FitnessGoal = FitnessGoal.MAINTAIN


@dataclass
class MacroResult:
    strategy_name: str
    tdee_kcal:     float
    target_kcal:   float
    protein_g:     float
    carbs_g:       float
    fat_g:         float
    fiber_g:       float
    notes:         list[str]

    def macro_split_pct(self) -> dict:
        total = self.target_kcal
        return {
            "protein": round((self.protein_g * 4) / total * 100, 1),
            "carbs":   round((self.carbs_g * 4) / total * 100, 1),
            "fat":     round((self.fat_g * 9) / total * 100, 1),
        }


class NutritionStrategy(ABC):
    """Interfaz base que TODA estrategia de dieta debe implementar."""

    @abstractmethod
    def calculate(self, bio: UserBiometrics) -> MacroResult:
        pass

    def _calculate_tdee(self, bio: UserBiometrics) -> float:
        """Mifflin-St Jeor si no hay % grasa. Katch-McArdle si hay % grasa."""
        if bio.body_fat_pct:
            # Katch-McArdle: más preciso para atletas
            lean_mass = bio.weight_kg * (1 - bio.body_fat_pct / 100)
            bmr = 370 + (21.6 * lean_mass)
        elif bio.sex == "M":
            bmr = (10 * bio.weight_kg) + (6.25 * bio.height_cm) - (5 * bio.age) + 5
        else:
            bmr = (10 * bio.weight_kg) + (6.25 * bio.height_cm) - (5 * bio.age) - 161
        return round(bmr * bio.activity_level, 0)

    def _apply_goal_adjustment(self, tdee: float, goal: FitnessGoal) -> float:
        adjustments = {
            FitnessGoal.BULK:       +300,
            FitnessGoal.CUT:        -400,
            FitnessGoal.MAINTAIN:      0,
            FitnessGoal.RECOMPOSE:     0,
        }
        return tdee + adjustments[goal]

class NutritionCalculator:
    """
    CONTEXTO del patrón Strategy.
    No conoce los detalles de ninguna estrategia concreta.
    Solo sabe que todas cumplen la interfaz NutritionStrategy.
    """

    def __init__(self, strategy: NutritionStrategy):
        self._strategy = strategy

    @classmethod
    def from_strategy_name(cls, strategy_name: str) -> "NutritionCalculator":
        """Factory method: crea la calculadora dada una string de estrategia."""
        StrategyClass = STRATEGY_REGISTRY.get(strategy_name.lower())
        if not StrategyClass:
            available = list(STRATEGY_REGISTRY.keys())
            raise ValueError(f"Estrategia '{strategy_name}' no registrada. Disponibles: {available}")
        return cls(StrategyClass())

    def set_strategy(self, strategy: NutritionStrategy) -> None:
        """Permite cambiar la estrategia en tiempo de ejecución (hot-swap)."""
        self._strategy = strategy

    def calculate_plan(self, bio: UserBiometrics) -> MacroResult:
        """Delega el cálculo a la estrategia activa. La calculadora no sabe cuál es."""
        return self._strategy.calculate(bio)

    def compare_strategies(self, bio: UserBiometrics) -> dict[str, MacroResult]:
        """Calcula el plan con TODAS las estrategias para comparar (endpoint /compare)."""
        return {
            name: StrategyClass().calculate(bio)
            for name, StrategyClass in STRATEGY_REGISTRY.items()
        }

    try:
        return _hasher.verify(hashed, plain)
    except VerifyMismatchError:

# JWT RS256 — clave asimétrica (la pública puede compartirse con otros servicios)
def _load_private_key():
    key_data = os.environ["JWT_PRIVATE_KEY_PEM"]
    return serialization.load_pem_private_key(key_data.encode(), password=None)

def create_access_token(user_id: str, role: str) -> str:
    payload = {
        "sub":  user_id,
        "role": role,
        "iat":  datetime.now(timezone.utc),
        "exp":  datetime.now(timezone.utc) + timedelta(minutes=15),
        "iss":  "healthstack.api",
        "aud":  "healthstack.web",
    }
    return jwt.encode(payload, _load_private_key(), algorithm="RS256")

def verify_access_token(token: str) -> dict:
    public_key_pem = os.environ["JWT_PUBLIC_KEY_PEM"]
    try:
        return jwt.decode(
            token, public_key_pem,
            algorithms=["RS256"],
            audience="healthstack.web",
        )
    except jwt.ExpiredSignatureError:
        raise HTTPException(status.HTTP_401_UNAUTHORIZED, detail="Token expirado")
    except jwt.InvalidTokenError:
        raise HTTPException(status.HTTP_401_UNAUTHORIZED, detail="Token inválido")

def _sanitize_event(event: dict, hint: dict) -> dict | None:
    """Elimina datos sensibles antes de enviar a Sentry (AEPD compliance)."""
    SENSITIVE_KEYS = {
        "password", "password_hash", "health_uuid_enc",
        "health_subject_id", "weight_kg", "body_fat_pct",
        "email", "token", "authorization",
    }
    def scrub(obj):
        if isinstance(obj, dict):
            return {
                k: "[REDACTED]" if k.lower() in SENSITIVE_KEYS else scrub(v)
                for k, v in obj.items()
            }
        elif isinstance(obj, list):
            return [scrub(i) for i in obj]
        return obj
    return scrub(event)

    start = time.time()
    response = await call_next(request)

Log estructurado con Loki. La tabla audit_log (pendiente de implementar) debe registrar: quién accedió a qué datos de salud y cuándo, sin incluir el contenido de los datos.

📋 Requerido
Art. 35 — Evaluación de Impacto (DPIA)

Obligatoria para plataformas que traten datos de salud a gran escala. Debe realizarse antes del despliegue en producción. Contactar con el delegado de protección de datos (DPD) designado.

⚠️
Nota legal importante: Esta arquitectura implementa medidas técnicas de seudonimización conformes al RGPD, pero la plataforma debe complementarse con una Política de Privacidad actualizada, un Aviso de Cookies, el nombramiento de un DPD (Delegado de Protección de Datos) y la realización de una DPIA si el tratamiento de datos de salud supera los umbrales de riesgo alto. Consulta con un especialista legal en privacidad antes del lanzamiento.
